# Week 6 - Data Modeling & Data Access Concepts

## Overview
This week focused on understanding the fundamentals of data modeling and different methods of accessing and loading data. The sessions helped build a strong foundation in how data is structured and how it can be collected from various sources for analysis.   


## Day-wise Learning Summary

### Monday to Wednesday: Data Modeling & Data Access

- Learned the basics of data modeling and its importance in organizing data  
- Understood Fact Tables, which store measurable numerical data like sales and quantity  
- Explored Dimension Tables, which contain descriptive information like product details and customer data  
- Learned about Star Schema and Snowflake Schema and their differences in structure and complexity  
- Gained knowledge about APIs and how they help in connecting client and server to fetch real-time data  
- Explored different methods of loading datasets:
  - Using CSV files from the web  
  - Fetching data from databases  
  - Accessing data using APIs  
- Understood the concept of Primary Keys and how they uniquely identify records  
- Learned about Cardinality and how relationships between tables are defined (one-to-one, one-to-many)  
- Discussed limitations in accessing data from websites and the concept of web scraping with permissions  


### Friday: Review & Reflection Session

- Reviewed all the concepts learned during the week  
- Discussed real-time applications of data modeling and APIs  
- Clarified doubts related to schema design, API usage, and data access  
- Shared individual learnings and understanding of the topics  
- Improved communication and presentation skills through discussion  


## Key Outcomes

- Developed a clear understanding of data modeling concepts  
- Learned how to differentiate between fact and dimension tables  
- Gained knowledge of schema design (star vs snowflake)  
- Understood how APIs work in real-time data fetching  
- Explored multiple ways to load and access datasets  
- Improved understanding of database keys and relationships  
- Built confidence in explaining technical concepts
