# Week 5 - Understanding Data Types, EDA, and Dashboard Planning

## Overview
This week focused on understanding the dataset structure and learning key data analysis concepts before building dashboards. We learned about different types of data, the concept of Exploratory Data Analysis (EDA), and the importance of identifying outliers. Based on this understanding, we also defined analytical questions and planned the dashboards for our project.


## Day-wise Learning Summary

### Monday to Wednesday: Data Types and EDA Concepts

- Learned about different types of data used in data analysis
- Understood **categorical data**, **numerical data**, and **time-based (time-series) data**
- Identified these data types in our project dataset
- Learned about **Exploratory Data Analysis (EDA)** and its role in understanding the dataset
- Understood how EDA helps in identifying patterns, relationships, and trends in the data
- Learned about **outliers**, which are unusual or extreme values in the dataset that may affect analysis results

### Thursday: Analytical Question Framing

- Defined **five analytical questions** based on the dataset to guide our analysis
- The questions focus on support efficiency, ticket resolution time, and customer satisfaction
- These questions help in deciding the type of visualizations required for the dashboard

### Friday: Review and Reflection Session

- Each team member shared what they **learned and understood during the week**
- Everyone presented the **five analytical questions** they framed based on their dataset
- Participants also discussed **areas where they need improvement**
- The session helped reinforce the concepts of **data types, EDA, and analytical thinking**
- Feedback was provided on the questions and dashboard planning


## Key Outcomes

- Improved understanding of **different data types in datasets**
- Learned the importance of **Exploratory Data Analysis (EDA)** before visualization
- Understood the concept of **outliers and their impact on analysis**
- Developed **analytical thinking by framing key questions from the dataset**
- Planned the **dashboard structure** for visualizing insights effectively
