# Optimizing IT Support Team Performance Using Analytics  

## 1. Problem Statement  
IT support teams handle a high volume of service tickets daily, making it challenging to monitor performance, resolution efficiency, and workload distribution. Delays in resolving issues and uneven allocation of tickets can negatively impact service quality and customer satisfaction.  

This project focuses on analyzing IT support ticket data to identify operational gaps and improve overall efficiency. By cleaning and transforming raw data using Excel and Power Query, the goal is to enable meaningful analysis and support data-driven decision-making through an interactive dashboard.


## 2. Dataset Description  
The dataset used in this project contains IT service desk ticket information, including details related to ticket creation, assignment, priority, resolution, and customer feedback.  

### Key Fields:

| Column Name | Description |
|------------|------------|
| Ticket ID | Unique identifier for each support ticket |
| Issue Category | Type of issue reported (e.g., technical issue, system error, request) |
| Priority Level | Urgency of the ticket (Low, Medium, High, Critical) |
| Assigned Support Agent | Agent responsible for resolving the ticket |
| Ticket Status | Status of ticket (In Process, Closed) |
| Created Date | Date and time when ticket was created |
| Resolution Date | Date and time when ticket was resolved |
| Resolution Time | Time taken to resolve the issue |
| Customer Satisfaction Rating | Feedback rating given by users |

The dataset initially contained missing values and inconsistencies, which were handled using Excel and Power Query to ensure clean and structured data for analysis.


## 3. Key Performance Indicators (KPIs)  

- Total Tickets Handled – Overall volume of tickets processed  
- Average Resolution Time (hrs) – Efficiency of issue resolution  
- High Priority Ticket Count – Number of critical tickets requiring immediate attention  
- Tickets Resolved – Total number of successfully closed tickets  
- Resolution Rate (%) – Percentage of tickets resolved out of total tickets  
- Customer Satisfaction Score – Measure of user satisfaction  


## 4. Dashboard Description  

The dashboard provides a comprehensive overview of IT support operations through multiple visualizations:

- Department-wise Ticket Volume – Identifies which departments handle the most tickets  
- Priority Distribution – Shows how tickets are distributed across priority levels  
- Ticket Trend Over Time – Analyzes variation in ticket volume across time  
- Ticket Type Distribution – Highlights dominant ticket categories  
- Weekly & Daily Heatmap – Visualizes workload patterns across days and weeks  

The dashboard is designed with a clean layout, consistent color theme, and insight-driven titles to improve readability and usability.


## 5. Key Insights  

- Technical Support handles the highest number of tickets  
- Medium priority tickets form the majority of the workload  
- Ticket volume shows fluctuations over time, indicating varying workload patterns  
- Incident-type tickets contribute the most to overall ticket volume  
- Workload varies across different days and weeks, highlighting peak activity periods  
- Approximately 50% of tickets are resolved, indicating scope for improving efficiency  


## 6. Recommendations  

- Improve resolution efficiency by analyzing bottlenecks in unresolved tickets  
- Prioritize high-priority tickets to enhance service quality  
- Optimize workload distribution across departments and agents  
- Monitor peak periods and allocate resources accordingly  
- Focus on reducing resolution time to improve customer satisfaction  


## 7. Tools Used  

- Microsoft Excel  
- Power Query  
- Power BI  
- Data Cleaning & Transformation Techniques  


## Project Outcome  

This project demonstrates how raw IT support data can be transformed into actionable insights using data analytics tools. The dashboard enables stakeholders to monitor performance, identify inefficiencies, and make informed decisions to improve IT service operations.
